Examples

Lewinsky works on a simple mail interface. You simply need to mail your notes to note@mylewinsky.com. Lewinsky parses and stores your note for you. A note is the basic entity in Lewinsky.


Subject

Every note has a subject. The subject of the mail directly translates to the subject of the note with the exception of mail jargon like "Re:" "Fwd:" etc. So mail with a repeated subject will stack to the existing note with that subject.


Body

A note can contain a body. The body of the mail is parsed to remove special tokens (mentioned below) and the remainder becomes the body of the note. This note body is replayed whenever the note is recovered.


Actions

You can schedule actions on notes. Scheduling an action needs two special tokens "@date" and "@action". The "@date" token needs to be followed by a valid date time in the format YYYY-MM-ddTHH:mm. This will be the date time for the action scheduled on the note. The "@action" token needs to be follwed by one of the valid actions : "mail", "call" or "msg". There can be multiple "@action" tokens.

mail - The mail action looks for valid email ids in the recipient list (see below). In case no valid recipient is specified, it mails back to the creator of the note.
call, msg - The call and msg actions look for valid phone numbers in the recipient list (see below). A valid phone number is the complete number with international code. In case no valid phone number is provided, there is no action performed.

In case you want to cancel a scheduled action you can mantion a valid action to cancel after a "@cancel" special token.


Repetition

In case you want to schedule recurring actions, you can set a repeat pattern. The repeat pattern can be specified after a "@repeat" special token and can have the following self explanatory values : yearly / monthly / weekly / daily / hourly.
In case of repeated actions it is always the up to date note that will be replayed.


Recipients

Scheduled actions can have valid recipients for the actions in the form of email ids (mail action) or phone numbers (call and msg actions). The recipient can be specified after a "@to" special token.
